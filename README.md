# I-SEM Electricity Market & Physical Grid Data Pipeline

## High-Level Overview
This project operates an end-to-end data engineering architecture that captures, harmonizes, and stores real-time market operations, physical grid dispatch metrics, and localized weather observations for the Integrated Single Electricity Market (I-SEM) in Ireland.

The core asset is a localized SQLite database (`irish_grid.db`) that maintains an unedited, hourly historical record of wholesale power settlements alongside synchronized boundary constraints and meteorological variables.

---

## Architecture Design & Data Flow
The pipeline decouples data ingestion from analysis to ensure system stability and optimize data retrieval speeds.

1. **Ingestion Engine (`fetch_data.py`)**: Executes daily incremental polls utilizing a rolling 3-day window to capture late-settling server updates. It connects to the ENTSO-E and Open-Meteo API servers.
2. **Storage Vault (`irish_grid.db`)**: A lightweight relational database operating a strict `PRIMARY KEY` constraint on hourly timestamps to prevent duplicate records and enforce data integrity via upsert procedures.
3. **Analytics Workbench (`analysis.ipynb`)**: An entirely offline environment that pulls directly from the local database for rapid query processing, statistical reporting, and visual distribution.

---

## Technical Database Schema
The main data asset is written to the `market_actuals` table. Every row represents an immutable hourly historical fact:

| Column Name | Data Type | Measurement Unit | Factual Description |
| :--- | :--- | :--- | :--- |
| `timestamp` (PK) | TEXT | `YYYY-MM-DD HH:MM:SS` | Unified UTC chronological time marker. |
| `price_eur_mwh` | REAL | € / MWh | Day-Ahead market-clearing wholesale electricity price. |
| `actual_wind_mw` | REAL | Megawatts (MW) | Actual physical electricity generated by the aggregate wind fleet. |
| `demand_mw` | REAL | Megawatts (MW) | Total system generation output deployed to balance live consumer load. |
| `net_gb_flow_mw` | REAL | Megawatts (MW) | Cross-border subsea interconnector flow (Positive = Import to IE, Negative = Export to GB). |
| `wind_speed_80m` | REAL | Meters per second (m/s) | Physical wind speed captured at average turbine hub height. |
| `temperature_2m` | REAL | Celsius (°C) | Ambient air temperature monitored at ground level. |
| `net_demand_mw` | REAL | Megawatts (MW) | Residual grid load remaining for conventional generation units ($Demand - Wind$). |

## Observed Market & Physical Constraints
The dataset observes several technical and physical operational boundaries unique to the Irish grid:

* **Temporal Synchronization**: Data streams are programmatically locked to a unified **UTC baseline**, resolving the inherent 1-hour reporting lag between physical dispatch actions and financial settlement windows.
* **Asset Inertia Boundary**: Interconnector telemetry utilizes a capped **forward-fill guardrail (`limit=3`)** to naturally mimic physical asset flow continuity during brief remote server packet losses.
* **Onshore Capacity Ceiling**: The system tracks fleet output against a static onshore generation capacity proxy of approximately **5,000 MW**.
* **System Stability Threshold (SNSP)**: System operations conform to EirGrid's strict **75% SNSP Limit**, dictating that non-synchronous energy ($Wind + Imports$) cannot exceed 75% of live demand without grid instability.

## Core Analytical Perspectives Available
The synchronized schema allows for immediate factual exploration across four core domains:
1. **Price Frequency Distribution**: Categorization of historical hours into explicit financial brackets to evaluate extreme spike or negative-pricing intervals.
2. **Interconnector Congestion Metrics**: Calculating the exact percentage of hours subsea cables spend maxed out at thermal capacities versus floating freely.
3. **Commercial Block Variance**: Segmenting market outcomes into strict commercial Peak (07:00–19:00 Weekdays) vs. Off-Peak blocks.
4. **Wind Penetration Fractions**: Real-time evaluation of the exact renewable share making up the total physical electron pool.
